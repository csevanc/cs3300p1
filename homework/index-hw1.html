Problem 3

Type coercion is when JS casts/converts one of the types in an expression to another type that is also in the expression so that the data types match. Thus, type coercion occurs when an expression/operation has different data types, and it can be subtle and unexpected.

1. Type coercion produces an answer that is unexpected when you call: "console.log(typeof(1 + []))" because it returns 'string'. This is unexpected because in python's case, it throws an error saying that you cannot add an 'int' and a 'list' as '+' is an unsupported operand for those types. This is also unexpected because I would have expected the outcome to be 'number' as I assumed that an empty list would be converted into a 0, so that 1 + [] = 1, where 1 is of type "number".

2. Type coercion produces an answer that is unexpected when you call: "console.log([] == ![])" because it returns 'true'. This is unexpected because in python's case, it throws a syntax error as you cannot put the logical NOT operator '!' in front of an empty list []. This is also unexpected because I would have expected the outcome to be 'false' as I assumed that since [] is usually casted as 0, false, or an empty string in many cases, this would mean that [] == ![] is equivalent to 0 == 1 or false == true, which both evaluate to 'false'.
